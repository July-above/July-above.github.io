dns隧道

  1. nano /etc/hosts 在里面添加
  2. ping dns.xxx.com -c 2 (攻击机和中间机都需要)
  3. 需要在中间机修改 /etc/dns2tcpd.conf
    1. 修改前
    2. 修改后
  4. 中间机启动dns隧道：dns2tcpd -f /etc/dns2tcpd.conf -F -d 3
  5. 如果报错 systemctl stop systemd-resolved 再启动
  6. kali：dns2tcpc -r ssh -k root -z dns.xxx.com 192.168.80.174 -l 8888 -c -d 3
  7. 访问本机的8888端口就是访问中间机的22号端口 ssh -p 8888 127.0.0.1
